<task type="checkpoint:human-verify" gate="blocking">
  <what-built>
Complete React 19 + Vite 6 SPA dashboard at web/ with:
- Three stat cards (Balance / Reserved / Liquid) pulled from GET /api/accounts/default
- "Can I Buy It?" check widget — idle input → loading spinner → animated verdict card
- Upcoming obligations list with sorted rows and footer total
- TanStack Query polling (30s background refresh, refetchOnWindowFocus)
- Motion animated verdict card (scale + fade + oklch color pulse)
- Sonner toast error notifications
- Go API serving all routes under /api/ prefix
- Built SPA embedded in the Go binary via go:embed
  </what-built>

  <how-to-verify>
**Prerequisites:** Ensure the CIBI API is running on port 8080 with a populated SQLite database.

Step 1 — Build the frontend and start the API:
```bash
cd /c/Projects/cibi-api/web && npm run build
cd /c/Projects/cibi-api && go run ./cmd/cibi-api/
```

Step 2 — Open http://localhost:8080 in a browser.

**Check A — Dashboard loads with real data:**
- The page shows the CIBI header wordmark
- Three stat cards appear with dollar amounts (not "$0.00" unless your balance is actually zero)
- The "Can I Buy It?" widget appears below the cards
- The "Upcoming Obligations" section appears below the widget

**Check B — "Can I Buy It?" flow:**
- Type `75` in the amount input (the `$` prefix should appear to the left)
- Click CHECK
- The button should briefly show a spinner while the call is in flight
- A verdict card should animate in (scale + fade) with either YES (green) or NO (red)
- The card should show purchasing power, buffer remaining, and a risk level badge (LOW/MEDIUM/HIGH/BLOCKED)
- Click "Check another" — the card should disappear and the input should return

**Check C — Mobile viewport:**
- Open DevTools → Toggle device toolbar → set width to 375px
- Verify the three stat cards stack vertically (not side by side)
- Verify the amount input and CHECK button are usable (at least 44px tall)
- No horizontal overflow

**Check D — Desktop viewport:**
- Set width to 1280px (or just resize the window wide)
- Verify the three stat cards appear side by side in a row
- Content should be centered with max-width ~672px

**Check E — Background polling:**
- Open DevTools → Network tab → filter by "accounts"
- Wait ~35 seconds
- Verify a GET /api/accounts/default request fires automatically without any user action

**Check F — Error toast (optional — skip if API is healthy):**
- Stop the API server (Ctrl+C)
- Wait for TanStack Query to retry (up to 3 attempts), then check that a toast appears:
  "Could not load financial data. Retrying in 30 seconds."
- Restart the API server

**Check G — Verify all manual checks from 05-VALIDATION.md:**
- Animation is visual and smooth (no jump-cut, no freeze)
- Mobile layout has no overflow or unusable inputs
- Network tab confirms polling behavior
  </how-to-verify>

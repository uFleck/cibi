# Gap Closure: Handle Missing Pay Schedule

**One-liner:** Added user-friendly error when pay schedule missing, created API/CLI/UI to configure it, and added seed script.

## What Was Built

### Task 1: Error code for missing pay schedule
Modified `internal/service/engine.go` to wrap `sql.ErrNoRows` as `ErrPayScheduleRequired`.
Modified `internal/handler/errors.go` to detect this and return friendly message with code.

### Task 2: PaySchedule API endpoint
Created `internal/handler/pay_schedule.go` with `POST /api/pay-schedule`.
Updated `routes.go` and `app.go` to wire the handler.

### Task 3: CLI command
Added `cibi-cli account set-pay-schedule` with `--frequency`, `--anchor-date`, `--day-of-month`, `--day-of-month-2`.

### Task 4: UI configuration
Created `PayScheduleForm.tsx` and `/settings` route.
Updated `CheckWidget` to show toast on `PAY_SCHEDULE_REQUIRED`.

### Task 5: Seed script
Created `cmd/seed/main.go` to populate default pay schedule.

## Verification

Run the seed:
```bash
go run ./cmd/seed/main.go
```

Start API and test:
```bash
go run ./cmd/cibi-api/
```

Test missing pay schedule:
```bash
curl -X POST http://localhost:8080/api/check -H "Content-Type: application/json" -d '{"amount": 50}'
# {"code":"PAY_SCHEDULE_REQUIRED","error":"You need to set up your pay schedule..."}
```

Test creating schedule and check:
```bash
curl -X POST http://localhost:8080/api/pay-schedule -H "Content-Type: application/json" \
  -d '{"frequency":"biweekly","anchor_date":"2024-01-01","day_of_month":1,"day_of_month_2":15}'
curl -X POST http://localhost:8080/api/check -H "Content-Type: application/json" -d '{"amount":50}'
# {"can_buy":true,...}
```

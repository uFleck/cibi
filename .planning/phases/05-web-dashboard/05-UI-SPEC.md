## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge padding, inline gaps within stat values |
| sm | 8px | Compact padding inside obligation list rows |
| md | 16px | Card internal padding, input/button padding |
| lg | 24px | Gap between stat cards, section spacing |
| xl | 32px | Gap between major page sections (cards → check widget → list) |
| 2xl | 48px | Top/bottom page padding on mobile |
| 3xl | 64px | Top/bottom page padding on desktop |

Exceptions:
- Touch targets (CHECK button, "Check another" button): minimum 44px height.
- Verdict card: 24px internal padding on all sides.

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 16px | 400 (regular) | 1.5 |
| Label | 14px | 400 (regular) | 1.4 |
| Heading | 20px | 600 (semibold) | 1.2 |
| Display | 28px | 600 (semibold) | 1.1 |

**Usage mapping:**
- Body (16px/400): obligation list row descriptions, button labels, input placeholder.
- Label (14px/400): stat card captions ("Current Balance", "Reserved", "Liquid"), obligation date and amount fields, footer total row.
- Heading (20px/600): stat card values (the dollar amounts), check widget section title ("Can I Buy It?"), obligations section title ("Upcoming Obligations").
- Display (28px/600): verdict YES/NO label on the verdict card.

**Source:** shadcn/ui defaults; sizes selected for readability at mobile viewport widths.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `hsl(var(--background))` — near-white `#fafafa` (light) / near-black `#09090b` (dark) | Page background, overall surface |
| Secondary (30%) | `hsl(var(--card))` — `#ffffff` (light) / `#18181b` (dark) | Stat cards, check widget container, obligations list container |
| Accent (10%) | `hsl(var(--primary))` — `#18181b` (light) / `#fafafa` (dark) | CHECK button, "Check another" button |
| Destructive | `hsl(var(--destructive))` — `#ef4444` | Error toast, NO verdict pulse background |

Accent reserved for: CHECK button fill, "Check another" button fill. No other interactive elements use accent fill.

**Semantic overrides (Tailwind v4 CSS `@theme` block in `web/src/index.css`):**

```css
@theme {
  --color-verdict-yes: oklch(0.65 0.17 142);  /* green-600 equivalent */
  --color-verdict-yes-tint: oklch(0.97 0.04 142);  /* soft green surface */
  --color-verdict-no: oklch(0.60 0.22 25);    /* red-600 equivalent */
  --color-verdict-no-tint: oklch(0.97 0.04 25);   /* soft red surface */
  --color-risk-low: oklch(0.65 0.17 142);     /* green */
  --color-risk-medium: oklch(0.72 0.19 70);   /* amber */
  --color-risk-high: oklch(0.65 0.20 40);     /* orange */
  --color-risk-blocked: oklch(0.60 0.22 25);  /* red */
}
```

**Risk level color mapping:**
- `LOW` → `--color-risk-low` (green text badge)
- `MEDIUM` → `--color-risk-medium` (amber text badge)
- `HIGH` → `--color-risk-high` (orange text badge)
- `BLOCKED` → `--color-risk-blocked` (red text badge)

## Interaction Contracts

### Stat Cards

- **Component:** shadcn `Card` with `CardHeader` (caption label) and `CardContent` (dollar value).
- **Card variant:** default (border + subtle shadow). No elevation override needed.
- **Border radius:** `rounded-xl` (shadcn default via `--radius`).
- **Amount format:** `Intl.NumberFormat('en-US', { style: 'currency', currency: 'USD' })`.
- **Liquid card color:** When `liquid <= 0`, value text color switches to `--color-risk-blocked` (red). When `liquid > 0`, inherits `--foreground`.

### Check Widget — Idle State

- Full-width `Card` container.
- `CardHeader`: heading "Can I Buy It?" at 20px/600.
- `CardContent`: row layout — `Input` (type="number", min="0", step="0.01", placeholder="0.00") + `Button` ("CHECK", default variant, accent fill).
- Input width: `flex-1`. Button width: fixed at content width.
- Amount input prefix: static `$` rendered as absolute-positioned label inside the input wrapper (not a shadcn `Input` adornment prop — add via wrapping `div` with `relative`).

### Check Widget — Loading State

- CHECK button: replaces text with `Loader2` icon from lucide-react (`animate-spin`, 16px). Button stays same size.
- Input: `disabled={true}`. Visual opacity via Tailwind `disabled:opacity-50`.
- No skeleton — API is sub-100ms locally (CONTEXT D-09).

### Check Widget — Verdict State

- Replaces idle section entirely (not a modal). Animated entry via Motion.
- **Entry animation (CONTEXT D-10):**
  - `initial={{ scale: 0.8, opacity: 0 }}`
  - `animate={{ scale: 1.0, opacity: 1 }}`
  - `transition={{ duration: 0.25, ease: 'easeOut' }}`
  - Background color pulse: animate from `--color-verdict-yes` / `--color-verdict-no` (vivid) → `--color-verdict-yes-tint` / `--color-verdict-no-tint` (soft) over 400ms using Motion keyframes on `backgroundColor`.
  - Total animation sequence: scale+opacity 0–250ms, color pulse 0–400ms. Both start simultaneously. Under 400ms total (CONTEXT D-10).
- **Verdict card contents (CONTEXT D-11):**
  - YES/NO display label: 28px/600, `--color-verdict-yes` or `--color-verdict-no`.
  - Purchasing power: "Purchasing power: $X.XX" — 16px/400.
  - Buffer remaining: "Buffer remaining: $X.XX" — 16px/400.
  - Risk level: `Badge` component, color from risk color map.
- **"Check another" button:** Below verdict card, `Button` variant="outline", full width.

### Obligations List

- `Card` container with `CardHeader` ("Upcoming Obligations", 20px/600) and `CardContent`.
- Rows: `flex` layout — description (`flex-1`, 16px/400), amount (14px/400, right-aligned), date (14px/400, right-aligned, format `MMM D` e.g. "Apr 15").
- Negative amounts: rendered as `-$X.XX` in `--color-risk-blocked` (red text). Zero/positive amounts: `--foreground`.
- No pagination. No empty skeleton rows. Empty state: inline message "No upcoming obligations." (14px/400, muted foreground).
- Footer: `Separator` + footer row "Total reserved: -$X.XX" (14px/600, right-aligned). Footer amount color: red (`--color-risk-blocked`).
- Sorted by `next_occurrence` ascending — sort applied client-side after API fetch.

### Error State

- **Mechanism:** Sonner toast (shadcn `sonner` component), positioned bottom-right.
- **When:** API call fails (network error or non-2xx response from `/api/check` or `/api/transactions` or `/api/accounts/default`).
- **Toast copy:** See Copywriting Contract below.
- **Auto-dismiss:** 5 seconds.
- **No inline error states** — the page data continues to show last-known-good values (TanStack Query stale data behavior).

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA | **CHECK** |
| Verdict YES label | **YES** |
| Verdict NO label | **NO** |
| Check widget heading | **Can I Buy It?** |
| Amount input placeholder | **0.00** |
| "Reset" action | **Check another** |
| Obligations section heading | **Upcoming Obligations** |
| Obligations empty state | **No upcoming obligations.** |
| Obligations footer label | **Total reserved:** |
| Error — API unreachable | **Could not reach the server. Make sure the CIBI API is running.** |
| Error — check failed | **Something went wrong running the check. Try again.** |
| Error — data load failed | **Could not load financial data. Retrying in 30 seconds.** |
| Page header wordmark | **CIBI** |
| Balance card caption | **Current Balance** |
| Reserved card caption | **Reserved** |
| Liquid card caption | **Liquid** |

**Money format:** `Intl.NumberFormat('en-US', { style: 'currency', currency: 'USD' })` — produces `$1,234.56`. Negative amounts prepend the `-` before `$` naturally via this formatter.

**Date format:** `new Date(isoString).toLocaleDateString('en-US', { month: 'short', day: 'numeric' })` — produces `Apr 15`.

**No destructive actions in this phase.** The dashboard is read-only + check-only. No delete, update, or irreversible operations are exposed.

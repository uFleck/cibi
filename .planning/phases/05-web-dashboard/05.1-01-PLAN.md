<objective>
Gap closure: Handle missing pay schedule gracefully and provide way to configure it via API, CLI, and UI.

Root cause: CanIBuyIt requires a PaySchedule row in the database, but there's no way to create one and errors are not user-friendly.
</objective>

<context>
@.planning/phases/05-web-dashboard/05-04-PLAN.md (shows the error)
</context>

<tasks>

<task status="completed">
### Task 1: Add error code for missing pay schedule ✓

In `internal/service/engine.go`, when `GetByAccountID` returns "sql: no rows in result set", wrap it with a user-friendly error that includes a code `PAY_SCHEDULE_REQUIRED`.

In `internal/handler/errors.go`, detect this code and return a friendly message:
```json
{"error": "You need to set up your pay schedule before using Can I Buy It. Use POST /api/pay-schedule to configure.", "code": "PAY_SCHEDULE_REQUIRED"}
```
</task>

<task status="completed">
### Task 2: Add PaySchedule API endpoint ✓

Created `internal/handler/pay_schedule.go` with:
- `POST /api/pay-schedule` — Create or update pay schedule for an account

Registered in `routes.go`.
</task>

<task status="completed">
### Task 3: Add CLI command for pay schedule ✓

Added `cibi-cli account set-pay-schedule` command with:
- `--frequency weekly|biweekly|monthly`
- `--anchor-date YYYY-MM-DD`
- `--day-of-month` and `--day-of-month-2` for biweekly
</task>

</task>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| HTTP client → /api/* | Authenticated CRUD; inputs bound and validated by Echo |
| Internet/Tailscale → /public/* | Unauthenticated; token is the only access control |
| Handler → Service layer | Handler must not bypass service; no direct repo calls from handlers |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-02-01 | Information Disclosure | GET /public/friend/:token | accept | Token is 128-bit random; no PII beyond name + balance visible; Tailscale network limits exposure; no auth required per CONTEXT.md locked decision |
| T-08-02-02 | Tampering | POST /api/peer-debts/:id/confirm | mitigate | Confirm endpoint is authenticated (same /api group); only user can call it — per CONTEXT.md locked decision. No friend-facing confirm route. |
| T-08-02-03 | Spoofing | Public token guessing | accept | 128-bit entropy via crypto/rand; brute force infeasible at personal-app scale |
| T-08-02-04 | Tampering | PUT /api/group-events/:id/participants | mitigate | SetParticipants is a full-replace operation — atomic DELETE + INSERT in one SQL transaction prevents partial state; validated at repo layer |
| T-08-02-05 | Elevation | /public/* bypassing auth | accept | By design (CONTEXT.md: friend links are read-only, no auth); confirm and write operations remain on authenticated /api routes only |
</threat_model>
